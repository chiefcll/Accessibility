Announcements Using aria-live and role="alert"

View the full library of solutions.

There are two ways to have the screen reader announce something. You can either use aria-live="polite" or role="alert".

Rules of Engagement

  1. The container with aria-live or role="alert" must already exist in the DOM. You can't create a container on the fly and expect the contents inside it to announce.
  2. Showing and hiding an aria-live or role="alert" container does not trigger an announcement. This is because the content inside the container hasn't changed.

Tips and tricks

  • Always keep an empty aria-live container and an empty role="alert" container at the bottom of your page and style it for screen-reader-text. Then it will always be there whenever you need to announce something.
  • When injecting content to be announced into these containers pull the content out a second later. That way:
    • If you need to say the exact same thing your content has truly changed which will trigger the announcement
    • That prior content won't be discovered later on if a person using a screen reader is arrowing around your page.
